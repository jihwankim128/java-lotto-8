## 기능 요구사항 정리

### 로또 구입 금액

* 로또 구입 금액을 입력받는다.
    * 로또 1장의 가격은 1,000원이다.
    * 1000원으로 나누어 떨어지지 않으면 예외

### 로또 당첨 번호

* 당첨 번호 추첨 시 중복되지 않는 숫자 6개와 보너스 번호 1개를 뽑는다.
    * 당첨 번호는 쉼표를 기준으로 6개 입력받는다.
        * 6개 아니라면 예외
    * 로또 번호의 숫자 범위는 1~45까지이다.
        * 아니라면 예외

### 로또 발행

* 구입 금액에 따라 몇개의 로또를 발행할 지 결정한다.
* 중복되지 않는 6개의 숫자로 로또를 발행한다.

### 로또 당첨

* 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다.
* 1등: 6개 번호 일치 / 2,000,000,000원
* 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
* 3등: 5개 번호 일치 / 1,500,000원
* 4등: 4개 번호 일치 / 50,000원
* 5등: 3개 번호 일치 / 5,000원

### 수익률 계산

* 로또 당첨 금액과 구입 금액으로 수익률을 계산한다.

### 예외 처리

* 잘못된 값을 입력
    * IllegalArgumentException을 발생
    * "[ERROR]"로 시작하는 에러 메시지를 출력
    * 예외 발생 부분부터 입력을 다시 받는다.
* Exception이 아닌 IllegalArgumentException, IllegalStateException 등과 같은 명확한 유형을 처리한다.

## TDD를 위한 기능별 단위 정리

전체 기능에 대해 완벽히 파악하고, TDD를 진행하는 것은 Test-Last에 가깝다.
그렇다고 Bottom-Up으로 TDD를 진행하기에는 조합까지 고려할 경험이 부족하다.
전체 흐름에 대해 필요한 객체만 정리한 뒤, TDD를 진행한다.

### 전체 흐름

1. 구입 금액을 입력받는다.
    * Money 객체 필요
2. 로또를 n장 구매한다. (로또 생성)
    * Lotto 객체 필요
    * LottoGenerator 객체 필요
    * Lottos 객체 필요
3. 당첨 번호와 보너스 번호를 입력받는다.
    * WinningNumbers 객체 필요
4. 당첨 확인을 한다.
    * WinningNumbers 객체 필요
    * Lotto 객체 필요
    * Rank 객체 필요
5. 수익금을 계산한다.
    * WinningConfirmation 객체 필요
    * Money 객체 필요
